# TPLab6
TPLab6

Проблемы:

    1. Были лишние переменные и проверки по типу discount, который был объявлен с стандартным значением и потом постоянно переопределялся
    2. Не были разграничены ответственности, из-за чего, собственно, основная функция была избыточно большая
    3. Использовались везде магические числа, строки
    4. Было неочевидная генерация переменной order_id (тяжело читалось)
    5. В целом код сложно читался
    6. Не были использованы удобночитаемые аналоги или готовые решения языка для разных частей кода (if else -> match case, использлование тернарного оператора
    и удобного однострочного генератора)

Рефакторинги:

    1. Были применены рефакторинги по принципу DRY, KISS, чтобы не усложнять код и не изменять логику его работы.
    2. Во многом важные магические числа и строки были заменены на поля классов, разделённые по логике их связи с задачами
    3. Были поделены части кода одной большой функции на маленькие со своими конкретными задачами ( у каждой по одной) и ответственностями

Что стало лучше:

    1. Стало проще и лучше читать код в целом, так как функции буквально делять код на логические блоки его выполнения и идут в хронологическом порядке.
    2. Некоторые куски кода были заменены на удобные для чтения аналоги прямиком из самого языка (простой синтаксический сахар)
    3. Код, я считаю, стал очень удобен для масштабирования и безопасен от нечаянных изменений и в целом удобен для обновления чего-либо, так как все дублирования были выведены в
    отдельные фунции и константы (кроме ключей запросов данных из некой API)
